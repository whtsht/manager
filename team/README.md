## コミットメッセージのルール

-   日本語で記述
-   シンプルに何をしたのかを書く
-   App サーバーの変更ならば先頭に`app: `と記述  
    例)

    ```
    app: M34 予定リスト取得処理の単体テストを実装
    ```

-   Web サーバーの変更ならば先頭に`web: `と記述  
    例)

    ```
    web: M7 カレンダー表示処理を実装
    ```

-   チームフォルダの変更ならば先頭に`team:`と記述  
    例)

    ```
    team: 進捗報告書を更新
    ```

-   その他の変更ならば先頭に`other: `と記述  
    例)

    ```
    other: pythonのCIを修正
    ```

## ブランチ運用ルール

-   開発は`dev`ブランチで行う
-   アプリケーションの公開は`main`ブランチのものを使う
